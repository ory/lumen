Here's how Flask configuration loading works in this file:

## Class Hierarchy

**`Config(dict)`** (line 50) ‚Äî extends Python's built-in `dict`, so all config values are stored as key-value pairs accessible with normal dict syntax (`app.config["DEBUG"]`). It takes a `root_path` (for resolving relative filenames) and optional `defaults` dict.

**`ConfigAttribute`** (line 20) ‚Äî a descriptor that proxies attribute access on `App` to `app.config[name]`. This is how `app.debug` transparently reads/writes `app.config["DEBUG"]`.

## Key Loading Methods

### 1. `from_object(obj)` ‚Äî line 218
The foundational method. Accepts a module, class, or import string. Iterates `dir(obj)` and copies **only uppercase attributes** into the config:
```python
for key in dir(obj):
    if key.isupper():
        self[key] = getattr(obj, key)
```
If `obj` is a string, it's imported first via `werkzeug.utils.import_string`.

### 2. `from_pyfile(filename)` ‚Äî line 187
Loads a `.py`/`.cfg` file by **compiling and executing** it into a temporary module, then delegates to `from_object`:
```python
d = types.ModuleType("config")
exec(compile(config_file.read(), filename, "exec"), d.__dict__)
self.from_object(d)
```
Filenames are resolved relative to `root_path`.

### 3. `from_file(filename, load=...)` ‚Äî line 256
Generic file loader for structured formats (JSON, TOML, etc.). You pass a `load` callable (e.g., `json.load`, `tomllib.load`). The file is opened and passed to the callable, then the result goes to `from_mapping`. The `text` parameter controls text vs binary mode.

### 4. `from_mapping(mapping)` ‚Äî line 304
Filters a dict/mapping to only uppercase keys:
```python
for key, value in mappings.items():
    if key.isupper():
        self[key] = value
```

### 5. `from_envvar(variable_name)` ‚Äî line 102
Reads an **environment variable whose value is a file path**, then delegates to `from_pyfile`. If the env var is unset, raises `RuntimeError` (unless `silent=True`).

### 6. `from_prefixed_env(prefix="FLASK")` ‚Äî line 126
Loads all env vars starting with `FLASK_` (or a custom prefix). Values are parsed through `json.loads` to auto-convert types. Supports **nested dicts** via double-underscore separation ‚Äî `FLASK_SQLALCHEMY__DATABASE_URI` sets `config["SQLALCHEMY"]["DATABASE_URI"]`.

## The Loading Chain

```
from_envvar(var)  ‚îÄ‚îÄ‚Üí  from_pyfile(path)  ‚îÄ‚îÄ‚Üí  from_object(module)  ‚îÄ‚îÄ‚Üí  self[KEY] = val
from_file(path, load)  ‚îÄ‚îÄ‚Üí  from_mapping(dict)  ‚îÄ‚îÄ‚Üí  self[KEY] = val
from_prefixed_env()  ‚îÄ‚îÄ‚Üí  self[KEY] = json.loads(val)
```

The common invariant across all methods: **only uppercase keys are stored**. This lets you mix temporary lowercase variables in config files without polluting the config.

### `get_namespace(prefix)` ‚Äî line 323
Utility to extract a subset of config by prefix (e.g., `get_namespace("IMAGE_STORE_")` returns `{"type": "fs", "path": "..."}` with the prefix stripped and keys lowercased by default).
