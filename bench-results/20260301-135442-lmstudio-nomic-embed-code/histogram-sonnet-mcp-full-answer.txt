The core of histogram bucket counting is in `histogramCounts.observe`. Here are the relevant function signatures and their roles:

**Primary counting method** — `prom_histogram.go:652`
```go
func (hc *histogramCounts) observe(v float64, bucket int, doSparse bool)
```
This handles both classic and native (sparse) histograms:
- **Classic buckets**: `atomic.AddUint64(&hc.buckets[bucket], 1)` — increments the pre-computed bucket index
- **Native/sparse buckets**: computes a bucket `key` using `math.Frexp` + schema-based scaling, then routes the value to positive, negative, or zero buckets

**Key logic for native histogram key computation:**
- For `schema > 0`: binary searches `nativeHistogramBounds` (fractional bounds) then offsets by exponent
- For `schema <= 0`: uses the raw exponent, adjusted by the schema's offset and shift

Bucket counts are stored as delta-encoded `int64` slices (`PositiveBuckets`, `NegativeBuckets`), and validation reconstructs the total by summing deltas and comparing to `h.Count`.

**Supporting type** — `histogram.go:481`

Used for iterating over classic fixed-width buckets.

In short: classic bucket index is pre-computed by the caller and passed in; native histogram bucket keys are computed on the fly from the value's floating-point representation and the active schema.
