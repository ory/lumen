## TSDB Compaction: End-to-End

### 1. The `Compactor` Interface (`compact.go:52-77`)

```go
type Compactor interface {
    Plan(dir string) ([]string, error)
    Write(dest string, b BlockReader, mint, maxt int64, base *BlockMeta) ([]ulid.ULID, error)
    Compact(dest string, dirs []string, open []*Block) ([]ulid.ULID, error)
}
```

Three methods:
- **`Plan`** â€” scans a directory and returns sets of block dirs that are ready to be compacted (can be called concurrently)
- **`Write`** â€” persists a single `BlockReader` (e.g., the in-memory Head) to disk as a new Level 1 block
- **`Compact`** â€” merges multiple existing on-disk blocks into one

### 2. `LeveledCompactor` (`compact.go:79-93`)

```go
type LeveledCompactor struct {
    ranges                      []int64   // time range boundaries per level
    chunkPool                   chunkenc.Pool
    mergeFunc                   storage.VerticalChunkSeriesMergeFunc
    blockExcludeFunc            BlockExcludeFilterFunc
    enableOverlappingCompaction bool
    // ...metrics, logger, ctx, etc.
}
```

Constructed via `NewLeveledCompactorWithOptions` (`compact.go:202-241`), defaulting `mergeFunc` to `CompactingChunkSeriesMerger(ChainedSeriesMerge)` and `maxBlockChunkSegmentSize` to `chunks.DefaultChunkSegmentSize`.

**`Plan` â†’ `plan` (`compact.go:248-328`):**

1. Reads all block dirs, loads their `BlockMeta`
2. First tries `selectOverlappingDirs` â€” if any blocks have overlapping time ranges, compact those first (vertical compaction)
3. Otherwise calls `selectDirs` (`compact.go:330-367`) â€” groups blocks by the configured `ranges` levels; picks a group if it spans a full range interval or all blocks in it predate the most recent block
4. Falls back to single-block tombstone rewrite: if `>5%` of series are tombstoned in a sufficiently old block, returns that block alone for rewriting

**`Compact` â†’ `CompactWithBlockPopulator` (`compact.go:483-584`):**

1. Opens each dir's `Block` (reuses already-open blocks to avoid double-loading index)
2. Calls `CompactBlockMetas` to merge metadata (min/max time, sources, level)
3. Calls `c.write(dest, meta, DefaultBlockPopulator{}, blocks...)` to stream-merge all series and write the new block
4. If the result has 0 samples, marks source blocks `Deletable` instead of writing
5. Returns the new `[]ulid.ULID`

**`Write` (`compact.go:586-636`):** Used for Headâ†’disk flushing. Creates a Level 1 block from a `BlockReader`, delegates to the same internal `c.write` path.

### 3. How `DB` Triggers Compaction

#### Background loop (`db.go:1175-1243`, `DB.run`)

```go
func (db *DB) run(ctx context.Context) {
    for {
        select {
        case <-time.After(db.opts.BlockReloadInterval):
            db.reloadBlocks()
            db.compactc <- struct{}{}   // nudge compaction
            db.head.mmapHeadChunks()
            // stale series threshold check â†’ CompactStaleHead()
        case <-db.compactc:
            if db.autoCompact {
                db.Compact(ctx)         // main compaction entry point
            }
        }
    }
}
```

A tick fires every `BlockReloadInterval`, reloads blocks, then signals `compactc`. The `compactc` handler calls `DB.Compact`.

#### `DB.Compact` (`db.go:1410-1506`) â€” the orchestrator

```go
func (db *DB) Compact(ctx context.Context) error {
    db.cmtx.Lock()          // serializes with deletions
    // ...

```

Three phases, in priority order:

**Phase 1 â€” Head compaction (highest priority):**
```
while head.compactable() && !waitingForCompactionDelay():
    rh := NewRangeHeadWithIsolationDisabled(head, mint, maxt-1)
    head.WaitForAppendersOverlapping(rh.MaxTime())
    db.compactHead(rh)         // â†’ compactor.Write(db.dir, rh, ...)
    lastBlockMaxt = maxt
```
`head.compactable()` returns true when the head's oldest chunk is at least `chunkRange/2` behind the current time. `compactHead` calls `compactor.Write` (persists the range to disk), then `reloadBlocks` + `head.truncateMemory`.

**Phase 2 â€” OOO Head compaction:**
```
if lastBlockMaxt != MinInt64:
    db.compactOOOHead(ctx)     // writes OOO blocks, truncates WBL/mmap refs
```
Only runs if out-of-order was ever enabled. Uses `NewOOOCompactionHead` to collect OOO chunks and writes them as separate blocks.

**Phase 3 â€” On-disk block compaction:**
```
db.compactBlocks()
```

#### `DB.compactBlocks` (`db.go:1719-1763`)

```go
for {
    if head.compactable() && !waitingForCompactionDelay() {
        // Head needs attention again; stop block compaction early
        break
    }
    plan, _ := db.compactor.Plan(db.dir)
    if len(plan) == 0 { break }
    uids, _ := db.compactor.Compact(db.dir, plan, db.blocks)
    db.reloadBlocks()
}
```

Loops: plan â†’ compact â†’ reload, until `Plan` returns nothing or the head becomes compactable again (Head always wins priority).

### Summary: Data Flow

```
WAL append â†’ Head (in-memory)
    â”‚
    â”‚  [head.compactable() + delay elapsed]
    â–Ľ
compactor.Write(Head range) â†’ Level-1 block on disk
    â”‚
    â”‚  [Plan finds â‰Ą2 blocks in same range level]
    â–Ľ
compactor.Compact(dirs) â†’ higher-level block on disk
    â”‚
    â–Ľ
reloadBlocks() â†’ deletes source blocks marked Deletable
                  + applies retention (drops old blocks)
```

Compaction levels are defined by `ranges []int64` (e.g., `[2h, 6h, 24h]`). Blocks get merged upward through levels until they're either deleted by retention or reach the max range.
