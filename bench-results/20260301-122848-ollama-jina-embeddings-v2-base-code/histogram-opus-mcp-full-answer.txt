Here's how histogram bucket counting works in this codebase:

**There are two bucket systems: classic (regular) and native (sparse/exponential).**

### Classic Buckets

Classic histograms use pre-defined upper bounds (e.g. `DefBuckets = {.005, .01, .025, .05, .1, .25, .5, 1, 2.5, 5, 10}`). On each observation, the bucket index is found and the count is atomically incremented:

```go
// prom_histogram.go:652
func (hc *histogramCounts) observe(v float64, bucket int, doSparse bool)
```

The key line is `atomic.AddUint64(&hc.buckets[bucket], 1)` — a simple atomic increment of the bucket matching the observed value.

When writing out metrics (`histogram.Write` at `prom_histogram.go:779`), bucket counts are converted to **cumulative** counts:

```go
var cumCount uint64
for i, upperBound := range h.upperBounds {
    cumCount += atomic.LoadUint64(&coldCounts.buckets[i])
    // each bucket reports cumCount (sum of all buckets <= upperBound)
}
```

### Native (Exponential) Buckets

Native histograms use a schema-based exponential bucketing system. The bucket index is computed from the observation value using `math.Frexp`:

```go
frac, exp := math.Frexp(math.Abs(v))
if schema > 0 {
    key = sort.SearchFloat64s(bounds, frac) + (exp-1)*len(bounds)
} else {
    key = exp
    if frac == 0.5 { key-- }
    offset := (1 << -schema) - 1
    key = (key + offset) >> -schema
}
```

Observations are stored in three places based on value:
- `v > zeroThreshold` → `nativeHistogramBucketsPositive` (sync.Map)
- `v < -zeroThreshold` → `nativeHistogramBucketsNegative` (sync.Map)
- otherwise → `nativeHistogramZeroBucket` (atomic uint64)

### Key Types for Iteration

- **`regularBucketIterator`** (`histogram.go:481`) — iterates classic buckets
- **`cumulativeBucketIterator`** (`histogram.go:530`) — iterates native histogram spans, decoding delta-encoded bucket counts into cumulative counts, handling empty buckets between spans

The `Histogram` struct (`histogram.go:35`) stores native bucket counts as **delta-encoded** `int64` slices (`PositiveBuckets`, `NegativeBuckets`), with `Span` structs defining which bucket indices are populated.
