There are four label matcher types, defined as an `iota` enum in `labels_matcher.go`:

**`MatchType` enum** (`labels_matcher.go:21-29`)
```go
type MatchType int

const (
    MatchEqual     MatchType = iota  // =
    MatchNotEqual                    // !=
    MatchRegexp                      // =~
    MatchNotRegexp                   // !~
)
```

**`Matcher` struct** (`labels_matcher.go:46-53`)
```go
type Matcher struct {
    Type  MatchType
    Name  string
    Value string

    re *FastRegexMatcher
}
```

**Constructor** (`labels_matcher.go:55-70`)
```go
func NewMatcher(t MatchType, n, v string) (*Matcher, error) {
    m := &Matcher{
        Type:  t,
        Name:  n,
        Value: v,
    }
    if t == MatchRegexp || t == MatchNotRegexp {
        re, err := NewFastRegexMatcher(v)
        if err != nil {
            return nil, err
        }
        m.re = re
    }
    return m, nil
}
```

Key points:
- `NewMatcher(t, name, value)` is the standard constructor; returns an error if the value is an invalid regex for `MatchRegexp`/`MatchNotRegexp`.
- For the two regex types (`=~`, `!~`), the value is compiled into a `FastRegexMatcher` and cached in the unexported `re` field.
- `MustNewMatcher` is a panic-on-error convenience wrapper, explicitly documented as test-only.
