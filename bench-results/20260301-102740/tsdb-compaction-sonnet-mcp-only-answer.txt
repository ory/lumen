Here's how TSDB compaction works end-to-end:

---

## The `Compactor` Interface (`compact.go:52`)

```go
type Compactor interface {
    Plan(dir string) ([]string, error)
    Write(dest string, b BlockReader, mint, maxt int64, base *BlockMeta) ([]ulid.ULID, error)
    Compact(dest string, dirs []string, open []*Block) ([]ulid.ULID, error)
}
```

Three responsibilities:
- **`Plan`** — scans on-disk block dirs and returns a set that *can* be compacted concurrently
- **`Write`** — persists a single `BlockReader` (e.g. the Head) into a new Level-1 block on disk
- **`Compact`** — merges multiple existing blocks into one new block

---

## `LeveledCompactor` (`compact.go:79`)

The sole built-in implementation. Key fields:

```go
type LeveledCompactor struct {
    ranges                      []int64   // e.g. [2h, 6h, 18h, 54h] — the leveling ladder
    mergeFunc                   storage.VerticalChunkSeriesMergeFunc
    enableOverlappingCompaction bool
    blockExcludeFunc            BlockExcludeFilterFunc
    // ...metrics, pool, logger, ctx
}
```

Created via `NewLeveledCompactorWithOptions` (`compact.go:202`) or the shorthand `NewLeveledCompactor` (`compact.go:195`). The `ranges` slice defines the compaction levels — each successive range is typically 3× the previous.

---

## Planning: `Plan` → `plan` → `selectDirs` / `selectOverlappingDirs`

```
Plan(dir)
  └─ reads blockDirs, loads each BlockMeta
  └─ plan([]dirMeta)
       ├─ sort by MinTime
       ├─ selectOverlappingDirs → any time-range overlaps? compact those first
       └─ selectDirs
            └─ splitByRange(ds, rangeLevel)  // group blocks into aligned time windows
               → pick a group that spans a full range interval, or is older than newest block
       └─ fallback: any block with >5% tombstone ratio
```

`selectDirs` (`compact.go:330`) iterates `ranges[1:]` (skips level-0) and uses `splitByRange` (`compact.go:396`) to bucket blocks into aligned windows. A group is selected if it either **spans the full interval** `maxt-mint == iv` or **sits entirely before the most recent block**. The most recently created block is always excluded from planning (gives a backup window).

`selectOverlappingDirs` (`compact.go:369`) takes priority: if any blocks have overlapping `[mint, maxt)` ranges, they're returned immediately for vertical compaction.

---

## Compacting: `Compact` → `CompactWithBlockPopulator`

```go
func (c *LeveledCompactor) Compact(dest string, dirs []string, open []*Block) ([]ulid.ULID, error) {
    return c.CompactWithBlockPopulator(dest, dirs, open, DefaultBlockPopulator{})
}
```

`CompactWithBlockPopulator` (`compact.go:489`):
1. Opens each dir as a `Block` (reusing already-open blocks from `open` to avoid double-loading index data)
2. Calls `CompactBlockMetas` to merge the metadata into one new `BlockMeta` with a fresh ULID
3. Calls `c.write(dest, meta, blockPopulator, blocks...)` — the actual merge + write
4. If the resulting block has 0 samples, marks all source blocks `Deletable` instead of writing

---

## Writing: `Write` (Head → Block)

```go
func (c *LeveledCompactor) Write(dest string, b BlockReader, mint, maxt int64, base *BlockMeta) ([]ulid.ULID, error)
```

Used exclusively for flushing the in-memory **Head** to a Level-1 block. Creates a `BlockMeta` with `Compaction.Level = 1`, then delegates to `c.write(...)`.

## DB Integration: How Compaction Is Triggered

### DB struct (`db.go:281`)

```go
type DB struct {
    compactor Compactor
    compactc  chan struct{}   // signal: "please compact"
    autoCompact bool
    // ...
}
```

### Background loop: `DB.run` (`db.go:1175`)

A goroutine runs a `select` loop:
- On a **block reload timer** tick → reloads blocks then signals `compactc`
- On **`compactc`** receive → calls `db.Compact(ctx)` if `autoCompact` is true (with exponential backoff on error)
- Separately watches stale-series ratio; if `numStaleSeries/numSeries >= threshold` and next compaction isn't imminent, calls `db.CompactStaleHead()`

### `DB.Compact` (`db.go:1410`)

The main compaction entry point, holding `db.cmtx`:

```
DB.Compact(ctx)
  ├─ loop while head.compactable():
  │    ├─ wait for compaction delay (optional, random jitter)
  │    ├─ wrap head in RangeHead [mint, maxt-1]
  │    └─ db.compactHead(rh)
  │         └─ compactor.Write(db.dir, rh, mint, maxt, ...)  // Head → disk block
  ├─ db.head.truncateWAL(lastBlockMaxt)
  ├─ if head was compacted: db.compactOOOHead(ctx)   // out-of-order data
  └─ db.compactBlocks()
       └─ loop:
            ├─ plan := compactor.Plan(db.dir)
            ├─ compactor.Compact(db.dir, plan, db.blocks)
            └─ db.reloadBlocks()   // scans dir, loads new block, drops deleted
```

`head.compactable()` (`head.go`) returns true when the head's time range exceeds half the chunk range — meaning there's at least one full chunk range of "cold" data ready to flush.

## End-to-End Summary

```
Ingestion → Head (in-memory WAL-backed)
                │
                │ head.compactable() == true
                ▼
         compactor.Write()  →  Level-1 block on disk  (2h window)
                │
                │ Plan() selects 3× Level-1 blocks filling a 6h window
                ▼
         compactor.Compact()  →  Level-2 block  (6h)
                │
                │ Plan() selects 3× Level-2 blocks
                ▼
         compactor.Compact()  →  Level-3 block  (18h)
                │  ...
                ▼
         Retention / reloadBlocks() deletes old blocks
```

The leveled ranges (e.g. `[2h, 6h, 18h, 54h]`) are set at `DB.Open` time. Overlapping blocks (from out-of-order ingestion or downstream systems) are always compacted first via vertical merge using `mergeFunc`.
