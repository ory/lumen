Here's how histogram bucket counting works in this codebase:

## Two-Layer Architecture

There are two histogram implementations: a **Prometheus client** (`prom_histogram.go`) and a **model type** (`histogram.go`).

---

## Core Function Signatures

**Finding the right bucket** (`prom_histogram.go:864`):
```go
func (h *histogram) findBucket(v float64) int
```
Uses linear search for arrays < 35 elements, binary search (`sort.SearchFloat64s`) for larger. Returns index into `h.upperBounds`, or `len(h.upperBounds)` for the +Inf bucket.

**Recording an observation** (`prom_histogram.go:899`):
```go
func (h *histogram) observe(v float64, bucket int)
```
Atomically increments `h.countAndHotIdx`. Uses a **hot/cold double-buffer** — two `histogramCounts` structs, index toggled via the MSB of the counter. Delegates to `hotCounts.observe()`, then calls `h.limitBuckets()` for sparse (native) histograms.

**Low-level counts update** (`prom_histogram.go:652`):
```go
func (hc *histogramCounts) observe(v float64, bucket int, doSparse bool)
```
- **Regular buckets**: `atomic.AddUint64(&hc.buckets[bucket], 1)` — direct array slot increment
- **Sparse/native buckets**: Computes a bucket key using `math.Frexp` + schema-dependent formula, then calls `addToBucket()` on a `sync.Map`
- Zero bucket: `atomic.AddUint64(&hc.nativeHistogramZeroBucket, 1)` for values within `zeroThreshold`

## Data Structures

```go
type histogramCounts struct {
    sumBits   uint64        // float64 sum stored as bits for atomic ops
    count     uint64

    nativeHistogramZeroBucket uint64   // zero bucket count
    buckets   []uint64                 // regular bucket counts (cumulative on Write)

    // Sparse native histogram buckets via sync.Map
    nativeHistogramBucketsPositive sync.Map  // key=bucket index, value=*int64
    nativeHistogramBucketsNegative sync.Map
}
```

## Read Path (`prom_histogram.go:779`)

```go
func (h *histogram) Write(out *dto.Metric) error
```
Flips the hot/cold index by adding `1<<63` atomically, then reads from the now-cold counts. Builds cumulative bucket counts by iterating `upperBounds` and accumulating: `cumCount += atomic.LoadUint64(&coldCounts.buckets[i])`.
