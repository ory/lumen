## Compactor Interface (`compact.go:52-77`)

```go
type Compactor interface {
    Plan(dir string) ([]string, error)
    Write(dest string, b BlockReader, mint, maxt int64, base *BlockMeta) ([]ulid.ULID, error)
    Compact(dest string, dirs []string, open []*Block) ([]ulid.ULID, error)
}
```

- **`Plan`** — surveys a directory and returns sets of block dirs that can be compacted concurrently (may be overlapping)
- **`Write`** — persists a `BlockReader` (e.g. head snapshot) into a new on-disk block; returns `[]ulid.ULID` (empty if 0 samples)
- **`Compact`** — merges/compacts a set of block dirs into one; caller must only pass dirs returned by `Plan`


### Key methods

| Method | Signature | Role |
|--------|-----------|------|
| `Plan` | `(dir string) ([]string, error)` | Public; delegates to `plan()` |
| `plan` | `(dms []dirMeta) ([]string, error)` | Selects dirs by level; calls `selectDirs` or `selectOverlappingDirs` |
| `selectDirs` | `(dms []dirMeta, ...) []dirMeta` | Non-overlapping block selection per level |
| `selectOverlappingDirs` | `(dms []dirMeta) []dirMeta` | Selects overlapping blocks for merging |
| `Compact` | `(dest string, dirs []string, open []*Block) ([]ulid.ULID, error)` | Delegates to `CompactWithBlockPopulator` |
| `CompactWithBlockPopulator` | `(dest, dirs, open, populator) ([]ulid.ULID, error)` | Opens blocks, merges series, calls `write` |
| `Write` | `(dest string, b BlockReader, mint, maxt int64, base *BlockMeta) ([]ulid.ULID, error)` | Wraps `write` for head → block path |
| `write` | `(dest string, b BlockReader, ...) ([]ulid.ULID, error)` | Creates chunk/index/tombstone writers; flushes to disk |

**Level selection logic**: blocks are grouped by time range. A group is compactable when it has ≥2 blocks at the same level, or a block's range is exceeded by accumulated samples.

## How DB Triggers Compaction (`db.go`)

### Entry point: `db.Compact()` (~line 1414)

Runs in a loop (called periodically by the compaction goroutine):

```
1. head.compactable()?  ──yes──▶  compactHead()
                                       │
                                       ▼
                              compactor.Write(head snapshot)
                              → new on-disk Block

2. reloadBlocks()  ──▶  refresh in-memory block list

3. compactOOOHead()  ──▶  compactor.Write(OOO head)
                           → one or more OOO Blocks

4. compactBlocks()
       │
       ▼
   compactor.Plan(dbDir)  ──▶  []dirs to compact
       │
       ▼
   compactor.Compact(dest, dirs, openBlocks)
       │
       ▼
   reloadBlocks() + delete old blocks
```

### Head compactability check (`head.go:1792-1801`)

```go
func (h *Head) compactable() bool {
    if !h.initialized() {
        return false
    }
    // Head spans > 1.5× chunk range → compact
    return h.MaxTime()-h.MinTime() > h.chunkRange.Load()/2*3
}
```

The 0.5× buffer preserves an appendable window while still flushing the older portion.

## Summary: End-to-End Flow

```
Head accumulates chunks
        │  (MaxTime - MinTime > 1.5 × chunkRange)
        ▼
db.Compact() → compactHead()
        │
        └──▶ compactor.Write(RangeHead) → Block L1 on disk

Periodic tick → compactBlocks()
        │
        └──▶ compactor.Plan()     → find compactable L1 dirs
        └──▶ compactor.Compact()  → merge L1 → L2 block
                │
                └──▶ LeveledCompactor.write()
                          │
                          ├── iterate merged series (ChunkSeriesMergeFunc)
                          ├── write chunks (chunkenc format)
                          ├── write inverted index (postings)
                          └── write tombstones
                          → atomic rename into place
```

Compaction is **leveled**: raw head flushes become L1, multiple L1s merge into L2, etc., controlled by the `ranges []int64` slice (e.g. `[2h, 6h, 24h, 1w]`).
