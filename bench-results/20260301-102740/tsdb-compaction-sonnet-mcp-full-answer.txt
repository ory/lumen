## TSDB Compaction: End-to-End

### 1. The `Compactor` Interface (`compact.go:54`)

```go
type Compactor interface {
    // Returns directories that can be compacted concurrently (may overlap).
    Plan(dir string) ([]string, error)

    // Persists one BlockReader into a new on-disk block.
    Write(dest string, b BlockReader, mint, maxt int64, base *BlockMeta) ([]ulid.ULID, error)

    // Merges multiple dirs/blocks into one. Dirs come from Plan().
    Compact(dest string, dirs []string, open []*Block) ([]ulid.ULID, error)
}
```

Three methods, three responsibilities: **plan → write (head) → compact (on-disk blocks)**.

---

### 2. `LeveledCompactor` (`compact.go:80`)

The only built-in implementation. Key fields:

```go
type LeveledCompactor struct {
    ranges                      []int64   // exponential time ranges, e.g. [2h, 6h, 18h, 54h]
    mergeFunc                   storage.VerticalChunkSeriesMergeFunc
    enableOverlappingCompaction bool
    postingsEncoder             index.PostingsEncoder
    // ...metrics, chunkPool, ctx, etc.
}
```

`ranges` is the hierarchy — blocks at level N cover `ranges[N]` milliseconds.

#### `Plan()` — what to compact (`compact.go:249`)

```
Plan(dir)
  └─ readMetaFile() for each block dir
  └─ plan(dms []dirMeta)
       ├─ sort by MinTime
       ├─ selectOverlappingDirs() — overlapping blocks have highest priority
       ├─ selectDirs()            — level-based: find blocks filling a full range
       └─ tombstone check         — blocks with >5% tombstones
```

- **Overlapping blocks** (OOO data): always compact first.
- **Level promotion**: `selectDirs` uses `splitByRange` to bucket blocks into aligned windows per range tier. If a bucket is full (spans the range or all blocks are older), those blocks get promoted to the next level.
- The **newest block is always excluded** from planning (gives a backup window).

#### `Compact()` — merge on-disk blocks (`compact.go:485`)

```go
func (c *LeveledCompactor) Compact(dest string, dirs []string, open []*Block) ([]ulid.ULID, error)
```

1. Opens each dir as a `Block` (reuses already-open ones to avoid double-loading index data).
2. Calls `CompactBlockMetas()` to merge metadata, bump compaction level, union sources.
3. Calls internal `write()` → `PopulateBlock()` → iterates sorted series, merges chunks, writes index + chunk files.
4. Atomically renames `<uid>.tmp` → `<uid>` when done.
5. If result has 0 samples: marks source blocks `Deletable`, returns `nil` (no new block).

#### `Write()` — persist the Head (`compact.go:586`)

Same internal `write()` path but wraps a `BlockReader` (the Head) instead of merging multiple blocks. Sets compaction level to 1.

### 3. The `DB` and How It Triggers Compaction

#### `DB` struct fields (`db.go:283`)

```go
type DB struct {
    compactor  Compactor          // the LeveledCompactor
    head       *Head              // in-memory write buffer
    compactc   chan struct{}       // signal channel: "please compact"
    cmtx       sync.Mutex         // serializes compaction + deletion
    autoCompact bool
    // ...
}
```

#### Trigger path 1: Appender commit (`db.go:1360`)

```go
func (a dbAppender) Commit() error {
    err := a.Appender.Commit()
    if a.db.head.compactable() {          // head spans > 1.5× chunkRange?
        select {
        case a.db.compactc <- struct{}{}: // non-blocking send
        default:
        }
    }
    return err
}
```

Every commit checks if the head is ready.

#### Trigger path 2: Background loop (`db.go:1175`)

```go
func (db *DB) run(ctx context.Context) {
    for {
        select {
        case <-time.After(db.opts.BlockReloadInterval):
            db.reloadBlocks()
            db.compactc <- struct{}{}  // periodic tick
        case <-db.compactc:
            db.metrics.compactionsTriggered.Inc()
            if db.autoCompact {
                db.Compact(ctx)
            }
        case <-db.stopc:
            return
        }
    }
}
```

#### `Head.compactable()` (`head.go:1795`)

```go
func (h *Head) compactable() bool {
    return h.MaxTime()-h.MinTime() > h.chunkRange.Load()/2*3
    // i.e., head spans > 1.5× chunkRange (the 0.5 is the appender buffer)
}
```

#### `DB.Compact()` — the orchestrator (`db.go:1414`)

```
Compact(ctx)
  ├─ Loop: while head.compactable() && delay elapsed:
  │     compactHead(RangeHead)
  │       └─ compactor.Write(dir, head, mint, maxt, nil)  ← HEAD → block
  │       └─ reloadBlocks()
  │       └─ head.truncateMemory()
  ├─ head.truncateWAL()
  ├─ compactOOOHead()                                      ← out-of-order
  └─ compactBlocks()
       └─ Loop:
             plan = compactor.Plan(dir)
             uids = compactor.Compact(dir, plan, db.blocks) ← merge on-disk
             reloadBlocks()
```

### Summary Flow

```
Appender.Commit()
    │  head spans > 1.5× chunkRange?
    ▼
compactc ──────────────────────► db.run() goroutine
                                      │
                                      ▼
                                  DB.Compact()
                                  ├─ compactHead()   → LeveledCompactor.Write()
                                  │                     (head → block, level=1)
                                  ├─ compactOOOHead()
                                  └─ compactBlocks()
                                       ├─ LeveledCompactor.Plan()
                                       │    selectOverlappingDirs | selectDirs | tombstones
                                       └─ LeveledCompactor.Compact()
                                            PopulateBlock(): merge series, write index+chunks
                                            atomic rename tmp → final dir
```

Key invariants:
- `cmtx` serializes all compaction and deletion — never concurrent.
- Empty result (0 samples) → source blocks marked `Deletable`, cleaned up on `reloadBlocks`.
- Block identity is a ULID; the DB path is determined by model/project hash (in agent-index) or just the data dir (in raw TSDB).
- Compaction level increments with each merge: L1 blocks = direct head flushes, L2+ = merged L1 blocks, etc.
