Here's a comprehensive explanation of how PromQL query evaluation works in the Prometheus codebase:

---

## PromQL Query Evaluation: Architecture Overview

```
NewInstantQuery / NewRangeQuery
        │
        ▼
   ParseExpr → PreprocessExpr (StepInvariantExpr wrapping)
        │
        ▼
    query.Exec(ctx)
        │
        ▼
    Engine.exec → execEvalStmt
        │           │
        │           ├── populateSeries (expand VectorSelectors)
        │           ├── construct evaluator
        │           └── evaluator.Eval(expr)
        │                       │
        │                       └── eval(expr) ← recursive type-switch dispatch
        │                                 │
        │                                 └── rangeEval (per-step loop)
        ▼
      *Result{Value, Warnings, Err}
```

## 3. Execution Call Chain

**Step 1** — Parse and preprocess (wraps step-invariant subexpressions):
```go
// engine.go:531
func (ng *Engine) NewInstantQuery(...) (Query, error) {
    pExpr, qry := ng.newQuery(q, qs, opts, ts, ts, 0)
    expr, err := ng.parser.ParseExpr(qs)
    *pExpr, err = PreprocessExpr(expr, ts, ts, 0)
```

**Step 2** — `query.Exec` delegates to `Engine.exec`:
```go

    return &Result{Err: err, Value: res, Warnings: warnings}
}
```

**Step 3** — `execEvalStmt` orchestrates everything:
```go
// engine.go:772
func (ng *Engine) execEvalStmt(ctx, query *query, s *parser.EvalStmt) (parser.Value, annotations.Annotations, error) {
    mint, maxt := FindMinMaxTime(s)
    querier, _ := query.queryable.Querier(mint, maxt)
    ng.populateSeries(ctxPrepare, querier, s)   // expand series into VectorSelectors

    ev := &evaluator{
        startTimestamp: timeMilliseconds(s.Start),
        endTimestamp:   timeMilliseconds(s.End),
        interval:       durationMilliseconds(s.Interval),
        lookbackDelta:  s.LookbackDelta,
        querier:        querier,
        // ...
    }
    val, warnings, err := ev.Eval(ctxInnerEval, s.Expr)
    // convert Matrix → Vector/Scalar as appropriate
    return val, warnings, err
}
```

**Step 4** — `evaluator.Eval` (public entry, handles cleanup):
```go
// engine.go:1189
func (ev *evaluator) Eval(ctx context.Context, expr parser.Expr) (v parser.Value, ws annotations.Annotations, err error) {
    defer ev.recover(expr, &ws, &err)
    v, ws = ev.eval(ctx, expr)
    if ev.enableDelayedNameRemoval {
        v = ev.cleanupMetricLabels(v)
    }
    return v, ws, nil
}
```

**`engine.go:1905`** — `eval` dispatches on concrete type:
```go
func (ev *evaluator) eval(ctx context.Context, expr parser.Expr) (parser.Value, annotations.Annotations) {
    switch e := expr.(type) {
    case *parser.AggregateExpr:    // sum, avg, count, topk, etc.
    case *parser.Call:             // function calls → FunctionCalls registry
    case *parser.BinaryExpr:       // +, -, *, /, ==, and, or, unless
    case *parser.VectorSelector:   // metric{label="val"}
    case *parser.MatrixSelector:   // metric[5m]
    case *parser.SubqueryExpr:     // expr[5m:1m]
    case *parser.NumberLiteral:    // 42
    case *parser.StringLiteral:    // "foo"
    case *parser.ParenExpr:        return ev.eval(ctx, e.Expr)  // transparent
    case *parser.UnaryExpr:        // -expr
    case *parser.StepInvariantExpr: // optimization: evaluate once, broadcast
    }
}
```

## 6. Alert and Recording Rule Evaluation

Both rule types implement a common `Rule` interface with an `Eval` method. They call back into the engine through a `QueryFunc`:

```go
// QueryFunc type (rules package)
type QueryFunc func(ctx context.Context, qs string, t time.Time) (promql.Vector, error)
```

**`alerting.go:382`** — `AlertingRule.Eval`:
```go
func (r *AlertingRule) Eval(ctx context.Context, queryOffset time.Duration, ts time.Time,
    query QueryFunc, externalURL *url.URL, limit int) (promql.Vector, error) {

    res, err := query(ctx, r.vector.String(), ts.Add(-queryOffset))
    // for each sample in res → create/update Alert in r.active map
    // state machine: Inactive → Pending → Firing
    //   Pending→Firing when ts.Sub(alert.ActiveAt) >= r.holdDuration
    //   StateFiring → kept for keepFiringFor after condition clears
}
```

**`recording.go:85`** — `RecordingRule.Eval`:
```go
func (rule *RecordingRule) Eval(ctx context.Context, queryOffset time.Duration, ts time.Time,
    query QueryFunc, _ *url.URL, limit int) (promql.Vector, error) {

    vector, err := query(ctx, rule.vector.String(), ts.Add(-queryOffset))
    // rewrite metric name and labels on each sample:
    for i := range vector {
        lb.Reset(sample.Metric)
        lb.Set(labels.MetricName, rule.name)
        rule.labels.Range(func(l labels.Label) { lb.Set(l.Name, l.Value) })
        sample.Metric = lb.Labels()
    }
    return vector, nil
}
```

**Alert state machine:**
```
Inactive ──(condition fires)──▶ Pending
Pending  ──(holdDuration met)──▶ Firing
Firing   ──(condition clears)──▶ Inactive  (or kept for keepFiringFor)
```

## Key Takeaways

| Concern | Mechanism |
|---|---|
| **Entry point** | `NewInstantQuery` / `NewRangeQuery` → parse → `PreprocessExpr` |
| **Execution** | `query.Exec` → `Engine.exec` → `execEvalStmt` |
| **AST dispatch** | `evaluator.eval` — recursive type-switch on `parser.Expr` |
| **Step iteration** | `rangeEval` — loops `startTimestamp..endTimestamp` by `interval` |
| **Function calls** | Global `FunctionCalls map[string]FunctionCall`; dispatched in `*parser.Call` case |
| **Rule evaluation** | `AlertingRule.Eval` / `RecordingRule.Eval` call `QueryFunc` → engine |
| **Optimization** | `StepInvariantExpr` wrapping avoids re-evaluating constant subexpressions per step |
