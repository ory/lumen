Here's how histogram bucket counting works in `prom_histogram.go`:

## Core Flow

**1. Entry point — public `Observe`** (`prom_histogram.go:766`)
```go
func (h *histogram) Observe(v float64)
```
Calls `findBucket` then `observe`.

**2. Find the bucket** (`prom_histogram.go:866`)
```go
func (h *histogram) findBucket(v float64) int
```
- Linear search for `n < 35` upper bounds (faster on small arrays)
- Binary search (`sort.SearchFloat64s`) for larger arrays
- Returns index into `upperBounds`, or `len(upperBounds)` for the `+Inf` bucket

**3. Lock-free observe dispatch** (`prom_histogram.go:900`)
```go
func (h *histogram) observe(v float64, bucket int)
```
- Uses a hot/cold double-buffer: `countAndHotIdx` MSB selects which `histogramCounts` is "hot"
- Atomically increments `countAndHotIdx` and routes to the hot counts
- Calls `limitBuckets` if sparse (native) histograms are enabled

**4. Actual counting** (`prom_histogram.go:655`)
```go
func (hc *histogramCounts) observe(v float64, bucket int, doSparse bool)
```
Three things happen atomically:
- `atomic.AddUint64(&hc.buckets[bucket], 1)` — classic fixed bucket increment
- `atomicAddFloat(&hc.sumBits, v)` — running sum update
- For **native/sparse histograms**: computes a logarithmic `key` via `math.Frexp`, then calls `addToBucket` on positive/negative sparse maps, or increments the zero bucket
- `atomic.AddUint64(&hc.count, 1)` last — signals observation is complete

**5. Sparse bucket limit enforcement** (`prom_histogram.go:919`)
```go
func (h *histogram) limitBuckets(counts *histogramCounts, value float64, bucket int)
```
If `nativeHistogramMaxBuckets` is exceeded, acquires a mutex and either widens the zero bucket (`maybeWidenZeroBucket`) or doubles bucket width (`doubleBucketWidth`).

## Key Design Choices

- **Lock-free hot path**: all observation writes use `atomic` ops; the mutex is only taken for bucket-limit enforcement
- **Hot/cold double buffer**: `Write` (for Prometheus scrape) swaps hot↔cold, collects cold atomically, then merges — no blocking during observation
- **Two histogram types coexist**: classic fixed-width buckets (`hc.buckets[]`) and sparse exponential buckets (`sync.Map` keyed by logarithmic index), both updated in the same `observe` call
