## PromQL Query Evaluation Architecture

### 1. Key Interfaces and Types

**`engine.go` — top-level interfaces:**

```go
// QueryEngine: the primary interface; implemented by *Engine
type QueryEngine interface {
    NewInstantQuery(ctx, q storage.Queryable, opts QueryOpts, qs string, ts time.Time) (Query, error)
    NewRangeQuery(ctx, q storage.Queryable, opts QueryOpts, qs string, start, end time.Time, interval time.Duration) (Query, error)
}

// Query: a prepared, executable query
type Query interface {
    Exec(ctx context.Context) *Result
    Close()
    Statement() parser.Statement
    Stats() *stats.Statistics
    Cancel()
    String() string
}
```

**`ast.go` — expression hierarchy:**

```go
type Node interface { String() string; Pretty(int) string; PositionRange() posrange.PositionRange }
type Expr interface { Node; Type() ValueType; PromQLExpr() }

// Concrete expression types:
type AggregateExpr struct { Op ItemType; Expr Expr; Param Expr; Grouping []string; Without bool }
type BinaryExpr   struct { Op ItemType; LHS, RHS Expr; VectorMatching *VectorMatching; ReturnBool bool }
type Call         struct { Func *Function; Args Expressions }
type MatrixSelector struct { VectorSelector Expr; Range time.Duration }
type VectorSelector struct { Name string; LabelMatchers []*labels.Matcher; Series []storage.Series; ... }
type SubqueryExpr  struct { Expr Expr; Range, Step time.Duration; Offset time.Duration; ... }
type StepInvariantExpr struct { Expr Expr } // engine optimization: eval once, reuse across steps
```

### 2. Query Lifecycle

**Instantiation (`engine.go:531–572`):**
```
NewInstantQuery / NewRangeQuery
    → parser.ParseExpr(qs)          // parse string → Expr AST
    → ng.validateOpts(expr)          // check @ modifier / negative offset flags
    → PreprocessExpr(expr, ...)      // wrap step-invariant subexpressions in StepInvariantExpr
    → returns *query (implements Query)
```

**Execution (`engine.go:248–256, 669–889`):**
```
query.Exec(ctx)
    → ng.exec(ctx, q)
        → ng.execEvalStmt(ctx, q, s)
            → querier := query.queryable.Querier(mint, maxt)  // open storage window
            → ng.populateSeries(ctx, querier, s)              // fill VectorSelector.Series
            → setOffsetForAtModifier(...)                     // adjust offsets for @ modifier
            → evaluator := &evaluator{start, end, interval, ...}
            → evaluator.Eval(ctx, s.Expr)                     // recursive AST evaluation
            → sort result Matrix
            → convert Matrix → Vector or Scalar if instant query
```

### 3. AST Node Evaluation — The `eval` Dispatch

The core is `(ev *evaluator) eval(ctx, expr)` at `engine.go:1905`, a type switch over every `Expr` node:

```go
func (ev *evaluator) eval(ctx context.Context, expr parser.Expr) (parser.Value, annotations.Annotations) {
    switch e := expr.(type) {

    case *parser.AggregateExpr:
        // eval e.Expr → Matrix, then rangeEvalAgg → group/reduce per step

    case *parser.Call:
        call := FunctionCalls[e.Func.Name]       // look up implementation
        // special-case: label_replace, label_join, info → dedicated methods
        // if no matrix arg: rangeEval → call(vectorVals, nil, args, enh)
        // if matrix arg: iterate over each series × each timestep, call(vectorVals, inMatrix, args, enh)

    case *parser.BinaryExpr:
        // scalar×scalar → scalarBinop
        // vector×vector → VectorAnd/Or/Unless/binop, depending on Op
        // vector×scalar or scalar×vector → rangeEval with scalar promotion

    case *parser.ParenExpr:  return ev.eval(ctx, e.Expr)

    case *parser.UnaryExpr:  // negate floats/histograms in the Matrix result

    case *parser.VectorSelector:  // populates via evaluateInstantVector
    case *parser.MatrixSelector:  // delegates to VectorSelector with range window
    case *parser.SubqueryExpr:    // evalSubquery → returns MatrixSelector
    case *parser.StepInvariantExpr: // eval once at start, replicate across steps
    case *parser.NumberLiteral, *parser.StringLiteral: // return as-is
    }
}
```

### 5. Alert Rules → Evaluation

**`alerting.go:382`:**
```go
func (r *AlertingRule) Eval(ctx context.Context, queryOffset time.Duration, ts time.Time,
    query QueryFunc, externalURL *url.URL, limit int) (promql.Vector, error) {

    res, err := query(ctx, r.vector.String(), ts.Add(-queryOffset))  // runs PromQL query
    // for each result sample:
    //   expand label/annotation templates
    //   create Alert{State: StatePending, ActiveAt: ts}
    //   store in r.active map keyed by label fingerprint

    // for each existing active alert:
    //   if still in results: keep / check if pending→firing (holdDuration elapsed)
    //   if not in results: mark inactive / apply keepFiringFor logic
    //   emit ALERTS{} and ALERTS_FOR_STATE{} time series samples
}
```

State machine: `StatePending` → `StateFiring` when `ts - a.ActiveAt >= r.holdDuration`

### 6. Recording Rules → Evaluation

**`recording.go:85`:**
```go
func (rule *RecordingRule) Eval(ctx context.Context, queryOffset time.Duration, ts time.Time,
    query QueryFunc, _ *url.URL, limit int) (promql.Vector, error) {

    vector, err := query(ctx, rule.vector.String(), ts.Add(-queryOffset))  // runs PromQL query

    // Override metric name and apply rule labels on every result sample:
    for i := range vector {
        lb.Reset(sample.Metric)
        lb.Set(labels.MetricName, rule.name)
        rule.labels.Range(func(l labels.Label) { lb.Set(l.Name, l.Value) })
        sample.Metric = lb.Labels()
    }
    // Check for duplicate labelsets after rewrite
    return vector, nil
}
```

Both rule types receive a `QueryFunc` — a callback that calls the engine — rather than holding the engine directly.

### 7. The Full Picture

```
HTTP API / Rule Manager
    │
    ├─ QueryFunc(ctx, qs, ts)
    │       │
    │       ▼
    │   Engine.NewInstantQuery → parser.ParseExpr → PreprocessExpr → query
    │   query.Exec(ctx)
    │       → execEvalStmt
    │           → populateSeries (storage lookup)
    │           → evaluator.Eval(ctx, s.Expr)
    │               → eval() type-switch on AST node
    │                   AggregateExpr → rangeEvalAgg
    │                   Call          → FunctionCalls[name](vals, matrix, args, enh)
    │                   BinaryExpr    → VectorAnd/Or/binop/scalarBinop
    │                   VectorSelector→ storage series scan
    │                   MatrixSelector→ range window from storage
    │
    ├─ AlertingRule.Eval → QueryFunc → state machine → ALERTS samples
    └─ RecordingRule.Eval → QueryFunc → label rewrite → new metric samples
```

Key design notes:
- `StepInvariantExpr` (engine optimization) wraps subexpressions whose value doesn't change per step (e.g. `time()`, scalar literals), so they're evaluated once.
- `rangeEval` is the workhorse that drives the step loop for instant-vector and scalar functions.
- Range-vector functions (`rate`, `avg_over_time`, etc.) bypass `rangeEval` and instead run their own inner loop over `ev.startTimestamp → ev.endTimestamp`, calling `matrixIterSlice` per series per step.
- `EvalNodeHelper.Out` is a pre-allocated output slice reused across steps to minimize GC pressure.
